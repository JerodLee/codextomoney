추천 로직 설명

후보군 생성부터 검증/보정/모델 확장까지 전체 흐름을 정리했습니다.

대시보드 추천 로직 운영 상태

1) 후보군 생성

빗썸 현물 + 비트겟 USDT-M 선물의 공통 심볼만 사용합니다.

롱: 빗썸/비트겟 24h 상승률 기준 숏: 비트겟 24h 하락률 + 펀딩 조건 유동성 바닥: 빗썸 거래대금 + 비트겟 거래량

2) 리스크 필터

과열/극단 펀딩/주문불가 종목을 제거해 변동성 리스크를 줄입니다.

과열 상한 보수 상한 절대 펀딩 상한 빗썸 주문 가능 여부

3) 모델 구조

  • v1: 기본 모멘텀/유동성/펀딩 점수
  • v2: 시장 추세(24h~1m), 펀딩 crowding, OI, 쏠림(regime) 보강
  • 활성 모델은 model_registry로 제어

4) 검증과 승률 집계

  • 추천 시점의 진입가를 저장
  • horizon 경과 후 blended 수익률 평가
  • 모델별 count/win_rate/avg/median을 별도 집계

5) 자동 보정

  • 저성과 시 보수적으로 필터 강화
  • 후보 고갈 시 완만하게 필터 완화
  • 보정 악화 시 롤백 + 쿨다운

6) 자동 모델 확장

  • 조건: 모델 평가수 24 이상 + 승률 45% 미만
  • LONG: momentum_long_v1 → momentum_long_v2
  • SHORT: momentum_short_v1 → momentum_short_v2
  • 확장 이벤트는 run_history/model_governance_events에 기록

7) 시장 상관도 해석 기준

  • 시장/BTC/ETH 추천상관은 추천방향(LONG=+1, SHORT=-1)과 시장 변화(1h 우선, 없으면 24h)의 상관계수입니다.
  • 따라서 ETH 추천상관은 ETH vs ETH 자기상관이 아니라, 추천 방향과 ETH 장세의 정합성을 뜻합니다.
  • 예: ETH 추천상관이 -0.17이면 최근 추천 방향이 ETH 장세와 약하게 반대로 움직였다는 의미입니다.
  • 표본(n)은 신뢰도 판단을 위해 유지합니다. n이 작으면 상관값 변동성이 커집니다.
