BITHUMB SPOT + BITGET FUTURES(USDT-M)

추천 로직 설명

현재 운영 중인 추천, 검증, 보정, 손실 경고 흐름을 요약했습니다.

대시보드로 돌아가기

1) 후보군 생성

빗썸 KRW 현물과 비트겟 USDT-M 선물의 공통 심볼만 대상으로 시작합니다.

빗썸 24h 등락률 ≥ 1.0% 비트겟 LONG 기준: 24h ≥ 1.0% 비트겟 SHORT 기준: 24h ≤ -1.0% SHORT 추가: 빗썸 24h ≤ 3.0% SHORT 추가: 펀딩비 ≥ -0.0005 빗썸 거래대금 ≥ 20억 KRW 비트겟 거래량 ≥ 1,000만 USDT

2) 안전 필터

과열 구간과 비정상 펀딩 구간을 제거해 급격한 역변동 리스크를 낮춥니다.

과열 상한: 40% 보수 상한: 20% |펀딩비| 상한: 0.0015 빗썸 주문 가능 여부 체크

3) 스코어링 및 추천

필터를 통과한 종목을 LONG/SHORT로 점수화해 상위 종목을 텔레그램으로 전달합니다.

  • 추천 메시지에 LONG/SHORT 방향을 함께 표시
  • 추천 목록에 펀딩비(Funding Rate), OI(Open Interest)를 함께 기록
  • SHORT는 비트겟 하락 모멘텀을 기준으로 우선 선별
  • 상위 추천에서 최소 1개 SHORT 슬롯을 보장(후보 존재 시)
  • 실행마다 새 후보를 계산하고 상위 N개를 추천
  • 추천 시점 가격(진입가)을 저장해 이후 성과를 추적

4) 성과 검증

추천 후 기본 15분 경과 시점에 방향(LONG/SHORT)을 반영한 수익률을 계산합니다.

  • LONG: 기존과 동일하게 빗썸/비트겟 수익률을 반영
  • SHORT: 비트겟 수익률을 우선 반전(하락 시 +수익)해 판단
  • 최종 블렌디드 수익률로 승/패 기록
  • 블렌디드 수익률이 0보다 크면 승(Win), 아니면 패(Lose)

5) 자동 보정(Auto Calibration)

최근 성과가 기준 이하이면 필터를 강화하고, 성과가 우수하면 탐색 폭을 소폭 확대합니다.

  • 후보가 너무 적으면 유동성/모멘텀 하한을 완화
  • 승률/수익률 저하 시 과열 상한·보수 상한 등을 보수적으로 조정
  • 보정 이벤트는 대시보드에서 전/후 승률 변화로 추적

6) 즉시 손실 경고

평가 대기 중인 추천 종목이 실시간 기준 음수 수익으로 내려가면 별도 경고를 보냅니다.

  • 기본 기준: 블렌디드 실시간 수익률 < 0%
  • 종목별 1회 경고로 중복 스팸 방지
  • 경고 메시지에 종목과 현재 수익률 표시

7) 시장 인디케이터

대시보드에서 시장 전체, BTC, ETH 장세를 다중 시간대로 추적하고 현재 쏠림(집중) 상태를 표시합니다.

  • 변동 구간: 24h / 12h / 6h / 1h / 15m / 5m / 1m
  • 시장 전체: 비트겟 선물 전체의 거래대금 가중 가격 변화를 기준으로 계산
  • BTC/ETH: 해당 심볼 가격 시계열 기준으로 동일 구간 변동 계산
  • 장세 판정: 1h 기준 +0.6% 이상 상승, -0.6% 이하 하락, 그 외 중립(1h 부재 시 24h 대체)
  • 쏠림 판별: BTC 주도 / ETH 주도 / 알트 전반 / 특정 알트 집중 / 균형

8) 추천-시장 관계/상관도

추천 방향(LONG/SHORT)과 시장 방향의 정합성을 현재/역사 기준으로 함께 보여줍니다.

  • 현재 관계: 비례/반비례/혼합/시장중립
  • 역사 상관도: 추천 방향 신호와 시장 방향 신호의 상관계수(-1~+1)
  • 추가: 종목별로도 동일한 상관도를 분리 계산해 표시
  • 양수에 가까울수록 전통적으로 비례, 음수에 가까울수록 반비례
